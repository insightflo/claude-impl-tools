## 현재 알고리즘 (v4.9.1)

```
ALGORITHM get_recommendation():

  # ① 복구 체크 (최우선) — COMPLETED는 정상 종료, 오탐 방지로 건너뜀
  IF .claude/orchestrate-state.json EXISTS AND STATE_STATUS IN ["RUNNING", "PAUSED"]:
    RETURN "/recover"
  IF git merge conflicts exist (git diff --diff-filter=U 결과 있음):
    RETURN "/recover"

  # ② 태스크 파일 체크
  IF TASKS.md NOT EXISTS:
    IF docs/planning/06-tasks.md EXISTS: RETURN "/tasks-migrate"
    ELSE: RETURN "/tasks-init"
  IF TASK_COUNT == 0:
    RETURN "/tasks-init"

  # ③ 유지보수 체크 (AGENT_COUNT=0 AND GOVERNANCE_DONE=no 인 경우만)
  IF source_code EXISTS AND AGENT_COUNT == 0 AND GOVERNANCE_DONE == "no":
    IF incomplete_tasks > 0: RETURN "/agile iterate"
    IF all_tasks_completed: RETURN "/audit"

  # ④ 거버넌스 체크 (미완료 태스크 있을 때만)
  IF TASK_COUNT >= 10 AND (DOMAIN_COUNT >= 2 OR TASK_COUNT >= 30) AND GOVERNANCE_DONE == "no" AND incomplete_tasks > 0:
    RETURN "/governance-setup"

  # ⑤ 인프라 체크 (거버넌스 완료 후, 미완료 태스크 있을 때만)
  IF GOVERNANCE_DONE == "yes" AND TASK_COUNT >= 30 AND AGENT_COUNT == 0 AND incomplete_tasks > 0:
    RETURN "project-team/install.sh --mode standard"

  # ⑥ 구현/배포 판단 (거버넌스 완료 + 인프라 준비)
  IF GOVERNANCE_DONE == "yes" AND AGENT_COUNT > 0:
    IF all_tasks_completed: RETURN "/audit"
    IF incomplete_tasks >= 80: RETURN "/orchestrate-standalone --mode=wave"
    IF incomplete_tasks >= 30: RETURN "/orchestrate-standalone"
    ELSE: RETURN "/agile auto"

  # ⑦ 소규모 신규 구현 (거버넌스 불필요, 코드 없음, 미완료 태스크 있음)
  IF TASK_COUNT > 0 AND TASK_COUNT < 30 AND incomplete_tasks > 0:
    RETURN "/agile auto"

  # ⑧ 완료
  IF all_tasks_completed:
    RETURN "/audit"
```

## 변수 정의
- STATE_STATUS: orchestrate-state.json의 "status" 필드값 (RUNNING/PAUSED/COMPLETED/none)
- TASK_COUNT: TASKS.md 전체 태스크 수
- incomplete_tasks: [ ] 미완료 태스크 수
- all_tasks_completed: TASK_COUNT > 0 AND incomplete_tasks == 0
- source_code EXISTS: src/ or app/ or lib/ 에 파일 존재
- AGENT_COUNT: .claude/agents/*.md 파일 수
- GOVERNANCE_DONE: management/project-plan.md 존재 여부 (yes/no)
